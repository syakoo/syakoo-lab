学生時代からずっとやりたかったゲーム作りを、一昨年くらいから初めてみました。

https://game.syakoo-lab.com/games/escape-from-the-workroom/

https://game.syakoo-lab.com/games/running-pudding/

ぜひこの記事を読んだ後に、やってみてください。

さて、２作品を今までに出してきたのですが、作るのは想像以上に大変でした。

## １作目: 「作業部屋からの脱出」

https://game.syakoo-lab.com/games/escape-from-the-workroom/

まずは「作業部屋からの脱出」です。

タイトルからわかるように、プレイヤーが作業部屋に閉じ込められ、部屋にある謎や仕掛けを解いて脱出するゲームです。

TypeScript での初めてのゲーム作りだったので、まずそこが大変だったのもそうですが、個人的に一番大変だったのは...

<Big>イラスト。</Big>

<Image
  src="/img/writings/20250302/escape-from-the-workroom.png"
  caption="作業部屋からの脱出 - 謎解きシーン"
  style={{ width: 500 }}
/>

ドット絵を始めて描いたのですがまあ勝手がわからなくて、仕事の合間を探して一つの絵に数日かかったりしました。

脱出ゲームもあってカットの数も多く、かなーり苦しめられました。

## ２作目: 「走るプリン」

https://game.syakoo-lab.com/games/running-pudding/

次に作ったのは「走るプリン」。

これは、楽しみにしていたプリンがトイレの間に食べられてしまい、容疑者の証言や手がかりを基に、プリンを食べた犯人を探す推理ゲームです。

前回より若干余裕ができ、絵もゲーム実装もこだわることができてきたのですが、前回と比べてかなり大変だったことがあります。それは...

<Big>キャラクター同士の会話。</Big>

<Image
  src="/img/writings/20250302/running-pudding.png"
  caption="走るプリン - キャラクター会話シーン"
  style={{ width: 500 }}
/>

完全に盲点でした。推理ゲームに会話は必要不可欠です。
ぶっちゃけ、キャラクターの名前も性格も頭の中のイメージのまま使っていたので、これではやばいと思い、一から考え直しました。

## おわりに

これまでのゲーム作りの中で大変だったことを日記のような形でまとめました。

せっかく TypeScript でゲームを作っているので、それにしかできないような仕掛けやゲーム性を考えていきたいです。
