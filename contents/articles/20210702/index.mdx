解きたい問題や作成したい機能を日々実装していると思いますが、その解き方、つまりアルゴリズムは無数に存在しています。
それらのアルゴリズムの性能を評価する方針として実行時間(時間計算量)や使用メモリ(空間計算量)の効率性があり、同じ結果を出すコードでも 1 週間以上かかるものもあれば一瞬で終わるものもあります。
コードの実行時間を大まかに求めることができれば、実行する前により早く終わるアルゴリズムを選択することができるようになります。

このシリーズでは、**アルゴリズムの時間計算量を解析する方法**を紹介します。
例や練習問題で用いるコードは Python を使用しています。

## 1. プログラムの実行時間

四則演算や値の代入、リストの要素の参照などの**基本的な処理**は一瞬で終わります。

試しに次のコードの実行時間を測ってみます:

```python
a = 1 + 2 - 3 * 4 / 5
xs = [1, 2, 3, 4, 5]
```

自分の環境で実行時間は約 $2.39 \times 10^{-7}$ 秒でした。
これらの基本的な処理はどれも人間の気にする範囲内ではほとんど同じ時間で終わるため、**それぞれの実行時間を $t$ としましょう**。
すると、このコード全体の実行時間 $T$ は $2t$ となります。

### 繰り返し処理の実行時間

次に、繰り返しの処理が入った関数を見てみます:

```python
def average(xs: list) -> float:
    sum_xs = 0
    for x in xs:
        sum_xs += x

    return sum_xs / len(xs)
```

各行は基本的な処理しかしていないため $t$ しかかかりませんが、`for` 文内は**配列 `xs` の長さ分処理を繰り返します**。
よって、この関数のそれぞれの行の実行時間は次のようになります:

```python
def average(xs: list) -> float:
    sum_xs = 0              # t
    for x in xs:            # t*len(xs)
        sum_xs += x         # t*len(xs)

    return sum_xs / len(xs) # t
```

そして、この配列 `xs` の長さを $N$ とすると、この関数の実行時間 $T(N)$ は次のようになります:

$$
T(N) = 2t + \sum_{i=1}^{N} 2t = 2t(N+1)
$$

先程の結果を用いると、$N$ が $10^7$ のとき $2$ 秒くらいかかることが推測できます。
実際はもっと少ない時間で計算は終わりますが、このように**基本的な処理を単位時間とすることによって実行する前に実行時間を推測することができます。**

### 組み込み関数の実行時間

最後に Python の組み込み関数の入った関数を見ていきます。

```python
def average2(xs: list) -> float:
    return sum(xs) / len(xs)
```

一見、繰り返し処理がないため実行時間は $t$ かのように思えますが、この `sum()` という組み込み関数は**配列の長さの計算時間が必要**であるため、
この関数の実行時間は $tN$ となります。(厳密には `sum()` の内部での実装によりますが、大体で構いません)

この `sum()` という関数は配列の総和をとる関数なので、配列の全ての要素を見る必要があるため $tN$ になるのは直感的にわかりますが、
Python の組み込み関数はイテレータなどによって直感に反するような計算時間のものがいくつかあります。
